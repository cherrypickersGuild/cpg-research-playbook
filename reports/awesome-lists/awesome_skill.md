## Synthesis

The "skill" ecosystem centers on Anthropic's Agent Skills format (`SKILL.md` with YAML frontmatter + markdown body), officially specified in `anthropics/skills` and documented across `code.claude.com`. In under a year the pattern has been adopted far beyond Anthropic: VoltAgent's aggregator alone catalogs skills from 40+ organizations (Microsoft, Cloudflare, Sentry, HashiCorp, OpenAI, Figma, Hugging Face, and more), spanning document automation, DevOps/infra guidance, security review, design-to-code, and API integration. Skills fill a specific niche in Claude Code's context model — loading only a name+description at session start and pulling in full instructions on demand — which official docs (Best Practices, How Claude Code Works, "Steering Claude Code") consistently position as the right home for *procedural, occasionally-needed* workflows, in contrast to always-loaded CLAUDE.md rules or fully isolated subagents. Community activity ranges from polished, well-maintained collections (claude-howto, 39.2k stars, actively synced to releases) to earnest but unproven experiments (cc-thinking-skills, which openly admits its reasoning-framework skills haven't yet demonstrated measurable benefit) — a useful reminder that skill *existence* doesn't imply skill *efficacy*. For an AX education program, the most immediately useful artifacts are: the official skill-creator/template skills (how to build one), the "Steering Claude Code" decision framework (when to build one), and domain-specific catalogs like AI Research Skills and Bloom (what a well-scoped skill suite looks like in practice).
